TimeLedger App

FRs - (Functional Requirements)

[] The user should be able to sign up
[] The user should be able to sign up with Google
[] The user should be able to log in
[] The user should be able the user to view their monthly hours history
[] The user should be able the user export their monthly hours to WhatsApp
[] The user should be able to see a monthly hours summary on the history page

BRs - (Business Rules)

[] The user should not be able to sign up with the same e-mail
[] The user should only be able to view monthly history from up to one year ago
[] The user can only have one monthly report per month (if they wants a new one, they need to delete the current report)

NFRs - (Non-Functional Requirements)

[] The user's password must be encrypted
[] The app data needs to be saved in a PostgreSQL database
